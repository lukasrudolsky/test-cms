# Project Analysis

Use this reference before designing the logo. The goal is to infer a useful creative brief from the repository without asking the user.

## Files To Inspect

- `package.json` for framework, scripts, and dependencies.
- README files for product positioning.
- app/layout, document, metadata, sitemap, manifest, and SEO helpers.
- homepage, navigation, hero, footer, and route components.
- content/CMS files, especially page copy and settings.
- CSS, Tailwind config, design tokens, theme utilities, and shared UI helpers.
- `public/` assets for favicon, icon, existing logo, and OG imagery.

Do not read, quote, or write secret values from `.env` files.

## Output

Save `docs/brand/project-analysis.md` with:

- verified facts;
- inferred assumptions;
- brand/product summary;
- current visual system;
- logo usage map;
- implementation targets.

Keep assumptions explicit and continue.
