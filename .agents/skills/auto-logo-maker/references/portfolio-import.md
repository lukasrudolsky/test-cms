# Portfolio Import

Use this reference before collecting public visual references.

## Compliance

- Check `robots.txt`, source terms, and page availability first.
- Download only files that are publicly visible without login, paywall, CAPTCHA, private APIs, or token bypass.
- Stay on the requested domain and direct asset/CDN domains used by that domain.
- Do not crawl social media or unrelated portfolio sites unless the user explicitly allows it.
- Use a modest request pace.
- Record source restrictions in `SOURCE.md`.

If terms restrict reuse, label the collection as internal visual reference only. Do not redistribute downloaded files or use them as final outputs.

## Filtering

Prefer logomarks, monograms, lettermarks, combination marks, monochrome variants, logo-focused brand presentations, and construction/geometric examples.

Reject social icons, navigation icons, portraits, office photos, decorative backgrounds, textures, unrelated illustrations, service icons, buttons, favicons, tracking pixels, low-resolution thumbnails when better originals exist, and near duplicates.

## Manifest

Each accepted item needs local filename, source image URL, source page URL, original filename, alt text, project title, dimensions, MIME type, file size, SHA-256, download date, inclusion reason, classification, and copyright/source note.

Maintain `exclude.txt`; future analysis must ignore filenames listed there.
