# Logo Rules

## Must

- Be original and explainable.
- Work in one color.
- Work at 16-24 px.
- Use simple construction and clean geometry.
- Be easy to redraw.
- Work as a favicon.
- Fit the current project design.
- Borrow principles from references without copying specific forms.
- Use valid, responsive SVG with `viewBox`.
- Use `currentColor` where practical.

## Must Not

- Copy a known brand or a provided reference.
- Use emoji, external icon libraries, raster images, base64, or `<image>`.
- Use random decoration or generic library-icon shapes.
- Be only a letter placed in a plain circle.
- Depend on gradients, filters, shadows, or effects for recognition.
- Use SVG `<text>` as the main symbol.
- Include JavaScript, empty paths, or unnecessarily complex geometry.

## Construction

Use precise coordinates, restrained element counts, and simple shapes or paths. Prefer balanced optical alignment over purely mathematical centering when the mark looks visually off.
